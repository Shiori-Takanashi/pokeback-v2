# tree

---

`tree` コマンドは、**ディレクトリ構造を階層的に表示する**ためのUnix系ツールです。視覚的に構造を把握できるため、プロジェクトやファイル群の概要をつかむのに便利です。

---

## 基本構文

```bash
tree [オプション] [ディレクトリ]
```

* ディレクトリを省略すると、現在のディレクトリ（`pwd`）が対象。

---

## 出力例

```bash
$ tree
.
├── main.py
├── config/
│   └── settings.py
└── utils/
    ├── helper.py
    └── logger.py

3 directories, 4 files
```

---

## 主なオプション

| オプション          | 説明                                              |
| -------------- | ----------------------------------------------- |
| `-L <n>`       | 階層レベルを制限（例: `-L 2` なら2階層まで）                     |
| `-f`           | フルパス表示                                          |
| `-d`           | ディレクトリのみ表示（ファイルは除外）                             |
| `-a`           | ドットファイル（隠しファイル）も表示                              |
| `-I <pattern>` | 除外したいファイル・ディレクトリをパターンで指定 |
| `--noreport`   | 最後の「x directories, y files」の統計を非表示              |
| `-o <file>`    | 出力をファイルに保存                                      |

---

## 注意点

* `tree` は多くのLinuxでは標準インストールされていないため、次で導入：

```bash
# Debian/Ubuntu
sudo apt install tree

# macOS (Homebrew)
brew install tree
```

---


## 適している用途

* ディレクトリ構造をレビューしたいとき
* Markdownで構造を示したいとき（出力をコピーして貼る）
* 自作CLIやビルド後の構成確認

## 限界

* 除外ルールが貧弱（`.gitignore`のような高度な指定ができない）
* フォーマット変換不可（JSON出力等には非対応）
* ソート順や柔軟な加工には不向き

---

## 補足：`tree` と `ls -R` の違い

* `tree`：視覚的で構造が明確。
* `ls -R`：機械的な再帰出力で視認性は低いがシンプル。
