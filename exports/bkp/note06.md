# git diff

以下は「`git diff`の基礎」から「VSCodeでの表示方法」までを体系的にまとめたマークダウン形式です。**「merge」という語は使用していません**。

---

> ## `git diff` の基礎と VSCode での表示方法

---

### > ## `git diff` の基本

`git diff` は **「変更の差分」を見るコマンド**。

```bash
git diff
```

これは、**現在の作業ディレクトリと最新のインデックス（`git add` した内容）との違い**を表示する。

---

### > ## 主な使用例

| コマンド                | 意味                                         |
| ------------------- | ------------------------------------------ |
| `git diff`          | **未ステージの変更**（ファイルを編集したが `git add` していない部分） |
| `git diff --staged` | **ステージ済みの変更**（`git add` した内容と最後のコミットとの差分）  |
| `git diff HEAD`     | **最新のコミットと作業中ディレクトリ**全体の差分（ステージ前後含む）       |
| `git diff ファイル名`    | 特定のファイルの変更内容を見る                            |

---

### > ## 差分の見方（出力例）

```diff
- print("hello")
+ print("hello world")
```

* `-` は削除された行
* `+` は追加された行

---

### > ## VSCode 上での差分確認方法

#### > ❶ サイドバー「Source Control」アイコンをクリック（または Ctrl + Shift + G）

変更されたファイル一覧が表示される。

#### > ❷ ファイル名をクリック

左右に分割された画面で「変更前」「変更後」が表示される。

#### > ❸ 色の意味

| 色      | 意味     |
| ------ | ------ |
| 緑色     | 追加された行 |
| 赤色     | 削除された行 |
| 青い線（枠） | 修正された行 |

---

### > ## 補足：ステージングの前後を確認する方法

```bash
git add ファイル名
git diff --staged
```

VSCodeでは、Source Control ペインの中でステージしたファイルも確認できる。ステージングされた差分は右側に表示される。

---

### > ## 変更を破棄したいとき

VSCode 上で：

* ファイル右クリック → **「変更の破棄」**
* CLIでは：`git restore ファイル名`

---

### > ## まとめ

* `git diff` は **編集内容の確認**に不可欠
* VSCode は視覚的に差分を確認できるため効率的
* ステージ済み・未ステージの状態で使い分ける
