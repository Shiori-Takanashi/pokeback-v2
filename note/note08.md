# パターンマッチング

`正規表現（regex）` と `glob` はどちらも**文字列のパターンマッチング**に使われますが、目的・表現力・文法が大きく異なります。以下、冷静に本質と違いを整理します。

---

> ## 🔍 概要比較表

| 観点     | 正規表現（regex）                  | glob（ワイルドカード）                         |                   |
| ------ | ---------------------------- | ------------------------------------- | ----------------- |
| 主な用途   | 文字列の**部分一致・抽出・置換**           | **ファイル名・パスの一致**                       |                   |
| 表現力    | 非常に高い（検索、抽出、変換に対応）           | 限定的（主にパス文字列の一致）                       |                   |
| 記述の複雑さ | 高い（学習コスト大）                   | 簡単（直感的）                               |                   |
| サポート記号 | `.` `*` `+` `?` `[]` `()` \` | \` など                                 | `*` `?` `[]`（限定的） |
| エスケープ  | 必要（`.` や `*` は特別な意味を持つ）      | 通常は少ないが `[]` 内などは注意が必要                |                   |
| 使用例    | `re.match()`, `re.sub()` など  | `ls *.py`, `gitignore`, `Path.glob()` |                   |

---

> ## 📌 glob の主な記号と意味

| 記号       | 意味                   | 例（マッチ対象）                            |
| -------- | -------------------- | ----------------------------------- |
| `*`      | 任意の文字列（0文字以上）        | `*.md` → `note.md`, `readme.md`     |
| `?`      | 任意の1文字               | `note?.md` → `note1.md`, `noteA.md` |
| `[abc]`  | `a`または`b`または`c`      | `note[12].md` → `note1.md`          |
| `[!abc]` | `a`, `b`, `c` 以外の1文字 | `note[!0].md` → `note1.md`, etc.    |

---

> ## 📌 正規表現の主な記号と意味（簡略）

| 記号       | 意味                | 例                           |     |              |
| -------- | ----------------- | --------------------------- | --- | ------------ |
| `.`      | 任意の1文字            | `a.b` → `aab`, `acb`        |     |              |
| `*`      | 直前のパターンの0回以上の繰り返し | `a*` → `""`, `a`, `aaaa`    |     |              |
| `+`      | 直前のパターンの1回以上の繰り返し | `a+` → `a`, `aa`, `aaa`     |     |              |
| `?`      | 直前のパターンの0回または1回   | `a?` → `""`, `a`            |     |              |
| `[...]`  | 指定した文字の1文字        | `[abc]` → `a`, `b`, `c`     |     |              |
| `(...)`  | グループ化             | `(ab)+` → `ab`, `abab`, ... |     |              |
| \`       | \`                | OR                          | \`a | b`→`a`, `b\` |
| `\d`     | 数字（\[0-9]）        | `\d+` → `123`, `4`          |     |              |
| `^`, `$` | 行頭・行末             | `^abc$` → `abc`のみ一致         |     |              |

---

> ## 実用的な違いの例

### glob

```bash
*.md           # Markdown ファイルすべて
note?.md       # note1.md ～ note9.md
note[123].md   # note1.md, note2.md, note3.md
```

### 正規表現（Python）

```python
import re

re.match(r"note\d1\.md", "note11.md")   # 下一桁が1のnoteファイルにマッチ
```

---

> ## 用途の整理

| 目的       | 使うべきもの                                  |
| -------- | --------------------------------------- |
| ファイル名の一致 | `glob`, `.gitignore`                    |
| ファイルの探索  | `Path.glob()`                           |
| 文字列処理・解析 | `re.match`, `re.search`, `re.sub`（正規表現） |

---

> ## 結論

* `.gitignore` や `Path.glob()` に使われるのは **glob パターン**
* **正規表現とは異なり、記述力が限定的**
* 正規表現の方が表現力は高いが、**globはパスに対して使うことに特化して簡素**
