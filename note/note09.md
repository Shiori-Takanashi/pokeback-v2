# git rm

---

`git rm` は、Git の追跡対象からファイルやディレクトリを削除するコマンド。
単なる `rm` ではなく、「削除したことを Git の履歴に記録する」ことが目的。

---

## 基本構文

* git rm \[オプション] ファイル名

---

## 主な用途

| 目的                 | コマンド例                   | 備考                       |
| ------------------ | ----------------------- | ------------------------ |
| Gitから削除し、物理的にも削除   | git rm file.md          | 通常の削除                    |
| Gitから削除するが、ファイルは残す | git rm --cached file.md | `.gitignore` と併用して再追跡を防ぐ |
| ディレクトリを再帰的に削除      | git rm -r notes/        | サブディレクトリも含めて削除           |
| 強制削除（変更がある場合でも削除）  | git rm -f file.md       | 未コミット変更も問答無用で削除          |
| ワイルドカードで複数ファイルを削除  | git rm \*.log           | 拡張子ベースなどでまとめて削除可能        |

---

## よくある使い方の比較

* `rm file && git add -u`
  → 結果は同じだが、操作の意図が不明確になりやすい

* `git rm file`
  → 「このファイルを削除した」という履歴が明示されるため推奨される

---

## 注意点

* `git rm --cached` はファイルを残すが、Gitの追跡を解除する。
* `.gitignore` に書いておかないと、再び追跡対象になる可能性がある。
* `git rm` しただけでは完結せず、削除の意図を `git commit` で記録する必要がある。

---

## ノートの一部だけ残す例

1. `.gitignore` にノート全体を追加

   * notes/

2. 下一桁が1のノートだけ除外解除

   * !notes/note?1.md

3. 不要なノートをGit追跡から外す（ファイルは残す）

   * git rm --cached notes/note92.md

4. コミット

   * git commit -m "一部ノートだけ残し、他はGit追跡から除外"

---

## 補足

* `git rm` は削除そのものより「削除操作の記録」が目的。
* `.gitignore` と `git rm --cached` は組み合わせて使う必要がある。
* 削除対象の明示により、履歴の可読性と意図の明確化が図れる。
